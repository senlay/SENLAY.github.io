# docker是什么？

Docker 是一个开源的应用容器引擎，基于 Go 语言 并遵从 Apache2.0 协议开源。 Docker 可以让开发者打包他们的应用以及依赖包到一个轻量级、可移植的容器中，然后发布到任何流行的 Linux 机器上，也可以实现虚拟化。 容器是完全使用沙箱机制，相互之间不会有任何接口（类似 iPhone 的 app）,更重要的是容器性能开销极低。目前有CE和EE版本，CE为社区版 EE为企业版

# linux下安装docker，并启动服务

  • 查看系统版本号以安装适配版本

uname -r

  • 建议首先更新yum源 yum update
  • 安装并运行Docker
    // 安装docker的依赖包
    sudo yum install -y yum-utils \
    device-mapper-persistent-data \
    lvm2
    
   // 设置稳定仓库
   sudo yum-config-manager \
       --add-repo \
       https://download.docker.com/linux/centos/docker-ce.repo
    
    // 安装最新版本   
    sudo yum install docker-ce docker-ce-cli containerd.io   
   
   //指定版本安装
   yum list docker-ce --showduplicates | sort -r
   sudo yum install docker-ce-<VERSION_STRING> docker-ce-cli-<VERSION_STRING> containerd.io
  • 配置镜像加速

修改etc/docker的 daemon.json文件，不存在就新建，添加镜像加速地址，如 { "registry-mirrors": ["https://qps9evve.mirror.aliyuncs.com"] } 配置完后需要手动更新服务： $ sudo systemctl daemon-reload $ sudo systemctl restart docker

  • 启动

sudo systemctl start docker

  • 验证是否安装成功

docker info 或者运行 hello-world 映像来验证是否正确安装了 Docker Engine-Community 。 sudo docker run hello-world

# docker卸载镜像

  • 先停止后卸载

docker ps -a 找出 "CONTAINER ID"
然后 停止它 docker stop c608062bf46f 删除它（进程干掉）：docker rm c608062bf46f docker images 找出镜像id 最后卸载该镜像 docker rmi 4ab4c602aa5e

# docker 命令

  • docker search 搜索的镜像名：搜索镜像
  • docker pull 镜像名：下载镜像
  • docker images ：查看镜像
  • docker rmi -f 镜像或者tag :删除镜像

-f 表示强制删除

  • docker image inspect java:8 查询原信息，包含安装路径等
  • docker查看启动日志

docker container ps查看运行的容器 docker container logs tag名字，如docker container logs tomcat

  • docker run --name tomcat -p 8888:8080 -d tomcat

-p指定端口号映射； -d表示后台运行； --name 给镜像起个名字，该名称不可重复以示唯一

  • docker exec ：在运行的容器中执行命令
  • 语法：docker exec [OPTIONS] CONTAINER COMMAND [ARG...]

    • OPTIONS说明：

      -d :分离模式: 在后台运行

      -i :即使没有附加也保持STDIN 打开

      -t :分配一个伪终端

exit退出当前docker容器
如： docker exec -it 镜像名称 /bin/bash

  • 如何将本地文件copy到docker镜像中

docker cp 本地路径 容器Id或name:容器目录 如：docker cp /usr/local/ssm.war tom:/usr/local/tomcat/webapps/ssm.war

  • 将docker内文件拷贝到本地文件夹内

docker cp 容器Id:本地路径 镜像路径 如：docker cp tom:/usr/local/tomcat/webapps/ssm.war /usr/local/ssm.war

  • docker基于已有镜像创建新的镜像

docker commit -m="描述" -a="作者" 容器ID或者tag 如：docker commit -m="my first image" -a="sky" tom:v1

# dockerfile 构建镜像

  • dockerfile是什么？

构建镜像的命令集

  • 如何创建？

首先需要编写Dockerfile文件 vim Dockerfile 如：

FROM hub.c.163.com/netease_comb/debian:7.9
MAINTAINER netease	

# 更新软件源
RUN echo "deb http://mirrors.163.com/debian/ wheezy main non-free contrib" >/etc/apt/sources.list
RUN echo "deb http://mirrors.163.com/debian/ wheezy-updates main non-free contrib" >>/etc/apt/sources.list
RUN echo "deb http://mirrors.163.com/debian/ wheezy-backports main non-free contrib" >>/etc/apt/sources.list
RUN echo "deb-src http://mirrors.163.com/debian/ wheezy main non-free contrib" >>/etc/apt/sources.list
RUN echo "deb-src http://mirrors.163.com/debian/ wheezy-updates main non-free contrib" >>/etc/apt/sources.list
RUN echo "deb-src http://mirrors.163.com/debian/ wheezy-backports main non-free contrib" >>/etc/apt/sources.list
RUN echo "deb http://mirrors.163.com/debian-security/ wheezy/updates main non-free contrib" >>/etc/apt/sources.list
RUN echo "deb-src http://mirrors.163.com/debian-security/ wheezy/updates main non-free contrib" >>/etc/apt/sources.list
RUN apt-get update	

# 安装常用软件
RUN apt-get install -y openssh-server apt-utils tomcat7
RUN rm -rf /var/lib/apt/lists/*	

# 设置允许root ssh远程登录
RUN sed -i s/"PermitRootLogin without-password"/"PermitRootLogin yes"/g /etc/ssh/sshd_config	

ADD docker-tomcat7 /etc/init.d/tomcat7
RUN chmod +x /etc/init.d/tomcat7	

# 容器需要开放SSH 22端口
EXPOSE 22
EXPOSE 8080	

# SSH终端服务器作为后台运行
ENTRYPOINT /etc/init.d/tomcat7 start && /usr/sbin/sshd -D
